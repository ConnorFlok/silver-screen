# 🎬 The Silver Screen Concierge

A golden-age Hollywood movie recommendation app powered by **MovieLens + TMDB** and built with **React + Vite + FastAPI**.

---


## Quick Start

### Step 1 — Generate ML artifacts

Open `CapstoneFinal_MovieLens.ipynb` in Google Colab or Jupyter and run all
cells. This trains the TF-IDF + K-Means models and saves the `artifacts/`
folder. Copy it into `backend/`:

```bash
cp -r /path/to/artifacts  silver-screen/backend/artifacts
```

### Step 2 — Start the backend

```bash
cd silver-screen/backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

You should see:
```
✅ Ready — 87,461 movies | 42,830 vocab terms | 7 clusters
```

Visit http://localhost:8000/docs for the interactive API explorer.

### Step 3 — Start the frontend

```bash
cd silver-screen
npm install
npm run dev
```

Open http://localhost:5173. When the backend is running, movie detail pages
show a **✦ ML Model Active** badge and use real TF-IDF similarity scores.
When the backend is offline the app falls back to local genre scoring
automatically — nothing breaks.

## ML Models

### TF-IDF Cosine Similarity
Each movie is represented as a vector over a vocabulary built from its
**genre + keyword + cast + director + overview** soup. Cosine similarity
between vectors gives content-based neighbours. The top-100 neighbours per
movie are pre-computed at training time and stored in
`cosine_topk_index.json` for O(1) lookup at serving time.

The `/similar/{id}` endpoint returns a `similarity` field (0–1) in each result.

### K-Means Clustering
Movies are grouped using a combined matrix of TF-IDF features + scaled
numeric signals (avg_rating, rating_count, year). Each movie belongs to one
cluster. **Silhouette score** measures how well a movie fits its cluster vs
neighbouring clusters — sorting by silhouette surfaces the most archetypal
members first.

The detail view shows both a **TF-IDF Similar** tab and a **Same Cluster** tab
when the backend is active.

## Project structure

```
silver-screen/
├── backend/
│   ├── main.py           ← FastAPI server
│   ├── recommender.py    ← CineScopeRecommender class

│   └── README.md
│
├── src/
│   ├── App.jsx
│   ├── utils/
│   │   ├── api.js        ← backend client with auto-fallback
│   │   ├── dataset.js    ← MovieLens pool + live TMDB merge
│   │   ├── tmdb.js       ← TMDB API functions
│   │   └── posters.js    ← poster URL + SVG fallback
│   ├── data/

│   │   └── search_index.json   ← 87,461 movies for local search
```

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `VITE_TMDB_API_KEY` | (required) | TMDB API key |
| `VITE_API_URL` | `http://localhost:8000` | Backend URL. Set to `disabled` to skip backend entirely. |
